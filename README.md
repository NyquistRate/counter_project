# Counter Project ŌĆö 4-bit Counter ASIC Front-End Flow

An end-to-end ASIC front-end design flow for a 4-bit synchronous up-counter ŌĆö from Verilog RTL, through logic synthesis with Cadence Genus, to IO padframe configuration for physical design.

## Design Overview

The core design is a 4-bit counter with an active-low asynchronous reset:

- **Clock (`clk`)** ŌĆö synchronizes counting
- **Reset (`rst`)** ŌĆö active-low; clears the counter to `0000` when asserted
- **Output (`out[3:0]`)** ŌĆö 4-bit count value, incremented on every rising clock edge while `rst` is high

## Repository Contents

| File | Description |
|---|---|
| `counter.v` | Verilog RTL source for the 4-bit counter |
| `counter_test.v` | Testbench (`tb_counter`) that drives `clk`/`rst` and simulates the counter |
| `synth_script.tcl` | Cadence Genus synthesis script ŌĆö reads the RTL, applies constraints, and synthesizes to the target standard cell library |
| `counter.sdc` | Synopsys Design Constraints file generated by Genus (clock definition, I/O delays, wire load model, cell exclusions) |
| `counter_netlist.v` | Gate-level netlist produced by synthesis, mapped to the `tsl18fs120_scl_ss` standard cell library |
| `padframe.io` | IO padframe configuration for physical design (pad placement around the chip periphery) |

## Design Flow

1. **RTL Simulation**
   `counter.v` is verified against `counter_test.v`, which toggles `clk` every 5 time units, holds `rst` low for an initial window, then releases it and lets the counter free-run.

2. **Logic Synthesis**
   `synth_script.tcl` drives Cadence Genus:
   - Reads and elaborates `counter.v`
   - Defines the clock (`clk`) and I/O delay constraints
   - Runs `syn_generic` ŌåÆ `syn_map` ŌåÆ `syn_opt` at high effort
   - Emits `counter_netlist.v` (gate-level netlist), `counter.sdc` (final constraints), and timing/area/power reports

3. **Synthesis Constraints** (`counter.sdc`)
   - Target library: `tsl18fs120_scl_ss` (slow-slow corner)
   - Clock period: 5 ps, defined on `clk`
   - Input delay on `clk`/`rst`, output delay on `out[3:0]`
   - `enclosed` wire load model
   - Specific library cells excluded via `set_dont_use`

4. **Gate-Level Netlist** (`counter_netlist.v`)
   Synthesized counter built from standard cells (`dfnrq4` flip-flops, `nd0x`/`nr0x`/`inv0d` logic gates) implementing the next-state logic for each output bit.

5. **Physical Design ŌĆö IO Padframe** (`padframe.io`)
   Defines the chip's pad ring: clock/reset input pads, power (`VDD`/`VSS`) and I/O power (`VDDO`/`VSSO`) pads, output pads for `out[0:3]`, additional I/O pads, and corner cells at all four corners, placed clockwise around the periphery.

## Getting Started

### Prerequisites
- A Verilog simulator (e.g., Icarus Verilog, ModelSim, or VCS)
- Cadence Genus (or a compatible synthesis tool) with access to the `tsl18fs120_scl_ss` standard cell library

### Simulate the RTL
```bash
iverilog -o counter_sim counter.v counter_test.v
vvp counter_sim
```

### Run Synthesis
Update the library and HDL search paths in `synth_script.tcl` to match your environment, then run in Genus:
```tcl
genus -files synth_script.tcl
```
This produces `counter_netlist.v`, an updated `counter.sdc`, and timing/area/power reports.

### Physical Design
Use `padframe.io` as the pad configuration input for your physical design / floorplanning tool to place the IO ring around the synthesized core.
